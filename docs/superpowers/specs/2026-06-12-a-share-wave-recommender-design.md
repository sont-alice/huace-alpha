# A股波段预测推荐软件设计规格

日期：2026-06-12

## 目标

构建一个本地 Web 软件，用于中国大陆 A 股 2-6 周波段推荐。软件不承诺固定收益或固定胜率，而是把“胜率足够高”落实为可验证的样本外回测指标、交易成本后表现、最大回撤、风险过滤和推荐解释。

第一版采用“回测 + 推荐一体”的产品方向。用户每天打开软件后，首先看到今日推荐列表；每一条推荐都必须能追溯到回测证据、特征贡献、风险标签和交易计划。

## 约束与原则

- 仅使用公开、合法、用户授权的数据源。
- 默认可无账号运行，使用 AKShare 获取基础 A 股数据。
- 支持配置 Tushare Pro token；存在 token 时优先使用 Tushare，缺失字段或权限不足时回退到 AKShare。
- 所有推荐都以概率评分和风险提示呈现，不写成收益保证。
- 没有通过回测门槛或数据质量检查时，系统允许输出“今日无推荐”。
- 第一版服务于研究和辅助决策，不接入真实交易下单。

## 产品范围

### 第一版包含

- 本地 Web 软件，推荐优先布局。
- 数据源配置页：AKShare 默认，Tushare Pro token 可选。
- 股票池管理：全 A、沪深主板、创业板、科创板、自定义排除列表。
- 数据更新：复权日线、基础信息、行业、财务摘要、资金流或可用近似字段。
- 特征工程：趋势、动量、波动、量价、资金、行业强度、估值和盈利质量。
- 模型训练：面向 2-6 周持有期的排序模型。
- 回测验证：滚动 walk-forward、交易成本、滑点、止损、分年度指标。
- 推荐输出：今日候选、评分、建议持有期、仓位上限、止损线、推荐理由、风险标签。
- HTML/CSV 导出：每日推荐和回测报告。

### 第一版不包含

- 自动实盘下单。
- 高频或分钟级交易。
- 非公开数据、内部消息、违规爬取或规避数据权限。
- 承诺必胜率、稳赚收益或保本描述。
- 复杂多账户组合管理。

## 用户界面

首屏采用“推荐优先”布局：

- 顶部指标：今日通过门槛数量、样本外胜率、最大回撤、最近一次数据更新时间。
- 主表：代码、名称、行业、评分、预期持有期、建议仓位上限、止损线、主要信号、风险标签。
- 侧栏：数据源状态、模型版本、回测门槛、过滤条件。
- 个股详情抽屉：K 线摘要、关键特征、历史相似样本表现、回测交易记录、失败案例。
- 导航页：今日推荐、回测证据、策略配置、股票池、数据设置。

## 系统架构

### 数据层

数据层提供统一接口，屏蔽 AKShare 和 Tushare Pro 的差异：

- `DataProvider`：统一的行情、基础信息、财务、行业、资金接口。
- `AkshareProvider`：默认公开数据源。
- `TushareProvider`：token 存在时启用，并处理积分、频次、权限错误。
- `ProviderRouter`：根据字段、日期和权限选择最佳数据源，失败时回退。
- `LocalStore`：本地缓存原始数据和标准化数据，建议使用 SQLite 保存元数据、Parquet 保存宽表。

数据质量检查包括交易日连续性、复权价格有效性、成交额缺失、停牌日期、异常涨跌幅、财务字段发布日期和未来函数检查。

### 特征层

特征按交易日生成，只允许使用当日收盘前或已公开披露的数据：

- 趋势动量：20/60/120 日收益、均线斜率、相对强弱。
- 波动风险：历史波动、最大回撤、ATR、缺口风险。
- 量价结构：成交额分位、换手率、量价背离、突破确认。
- 资金与行业：主力资金或可用替代字段、行业相对强度、行业拥挤度。
- 基本面：估值分位、盈利增长、ROE、负债率、财报披露滞后处理。
- 市场状态：指数趋势、市场宽度、涨跌停数量、成交额环境。

### 模型层

第一版采用混合路线：

1. 规则过滤：剔除 ST、退市风险、新股早期、停牌、成交额不足、连续涨跌停、极端异常数据。
2. 排序模型：使用 LightGBM 或 HistGradientBoosting 类模型，预测未来 2-6 周相对收益或超额收益排序。
3. 校准与分层：把模型分数转为分位排名和置信等级，不直接展示为确定收益率。
4. 组合约束：控制单行业、单风格、单市值带集中度。

标签定义建议从 20 个交易日和 30 个交易日两个持有期开始，目标为未来持有期收益是否跑赢基准或股票池中位数。

### 回测层

回测必须与推荐逻辑共用同一套特征、过滤和排序代码：

- 时间切分：训练、验证、样本外；采用滚动 walk-forward。
- 交易规则：按推荐日次日开盘或 VWAP 近似成交，持有 20-30 个交易日或触发止损退出。
- 成本模型：佣金、印花税、滑点、无法成交处理。
- 指标：样本外胜率、年化收益、最大回撤、收益回撤比、换手率、分年度表现、行业暴露。
- 失败分析：统计高分但亏损的样本，记录共同风险特征。

默认推荐门槛：

- 样本外胜率高于基准胜率至少 5 个百分点。
- 样本外收益回撤比大于 1。
- 最大回撤不超过配置上限。
- 最近一个滚动窗口不过度劣化。
- 推荐股票满足流动性和风险过滤条件。

这些门槛可以配置，但系统必须清楚显示当前门槛。

## 数据流

1. 用户点击更新数据或每日自动运行。
2. ProviderRouter 从 Tushare Pro 或 AKShare 拉取数据。
3. LocalStore 保存原始数据和标准化表。
4. 数据质量检查生成可用股票池。
5. 特征层生成每日特征快照。
6. 模型层训练或加载最新模型，生成候选评分。
7. 回测层验证当前策略版本是否通过门槛。
8. 推荐层输出今日推荐或“今日无推荐”。
9. Web 前端展示推荐、解释、风险和证据。

## 错误处理

- 数据源不可用：显示具体数据源、接口、日期范围和回退结果。
- Tushare 权限不足：提示接口权限或频次问题，并尝试 AKShare 回退。
- 数据质量失败：阻止训练和推荐，列出失败字段和股票数量。
- 回测未通过：不展示买入推荐，只展示观察列表和失败原因。
- 模型训练失败：保留上一个可用模型，但标记模型日期和数据日期不一致。

## 技术选型

- 后端：Python，FastAPI 或 Streamlit。第一版建议 Streamlit 更快落地；如果需要复杂交互和后台任务，再切换 FastAPI + React。
- 数据处理：pandas、numpy、pyarrow、polars 可选。
- 模型：scikit-learn、LightGBM。
- 存储：SQLite + Parquet。
- 图表：Plotly。
- 配置：YAML 或 TOML，敏感 token 放 `.env`。

## 测试与验收

### 单元测试

- 数据标准化字段一致。
- 复权行情和交易日对齐正确。
- 特征不使用未来数据。
- 规则过滤覆盖 ST、新股、停牌、低成交额、涨跌停。
- 标签生成与持有期一致。

### 集成测试

- 无 Tushare token 时可使用 AKShare 跑通样例股票池。
- 有 Tushare token 时优先使用 Tushare，并在权限不足时回退。
- 回测与今日推荐使用同一策略代码。
- 交易成本改变会影响回测结果。
- 未通过门槛时不会输出买入推荐。

### 人工验收

- 能打开本地 Web 页面。
- 能更新数据并看到数据源状态。
- 能生成 2-6 周波段推荐。
- 每只推荐股票能看到推荐理由、风险标签和回测证据。
- 能导出推荐 CSV 和 HTML 报告。

## 合规与风险提示

软件定位为研究辅助工具，不构成投资建议。A 股市场受政策、流动性、突发事件、停复牌和涨跌停限制影响，历史回测不能保证未来表现。所有页面和报告应展示风险提示，并避免保证收益、稳赚、必胜等表述。

## 外部依据

- AKShare 文档列出 A 股股票数据、指数、宏观等数据字典，并说明其基于 Python 提供财经数据接口。
- Tushare Pro 官方文档说明可通过 Python SDK 或 HTTP API 获取数据，并存在 token、积分、频次和独立权限限制。
