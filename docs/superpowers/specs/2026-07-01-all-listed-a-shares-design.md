# 全部上市 A 股统一排名设计

## 目标

将固定 3000 股大池改为动态的全部当前上市 A 股池。系统每日从真实上市公司列表构建目标集合，所有目标股票使用相同历史、特征、模型和评分口径，最终在同一全市场股票池内计算分位和排名。

当前本地真实列表约有 4925 只股票，但该数字只是当日快照，实现不写死 4925 或其他上限。

## 证券范围

纳入：

- 上证主板 A 股。
- 深证主板 A 股。
- 创业板 A 股。
- 科创板 A 股。
- 北交所当前上市 A 股。

排除 B 股、ETF、LOF、其他基金、债券、可转债、指数和已退市证券。ST 和其他风险警示股仍可进入数据与评估层，但继续由现有风控规则禁止进入买入推荐。

## 动态股票池

`DataRequest` 增加明确的“全部当前上市 A 股”模式，公网推荐和每日快照默认启用。该模式不使用 `max_symbols` 截断；本地内部测试可继续使用小型数量请求。

每次更新先构建当日目标代码集合，再与上一版已验证快照比较：

- 交集股票：使用上一版 4 年真实历史作为基线，只增量更新最近 45 天。
- 新上市或新进入数据集的股票：拉取自上市日以来的真实历史。
- 不再出现于当日上市列表的股票：从新快照和新排名中移除，但不回写或破坏旧版快照。
- 当日接口失败但仍在上市列表的股票：使用上一版真实历史补齐，并记录失败数量。

目标集合按证券代码确定，不依赖 AKShare 或 Tushare 返回顺序，因此不同机器不会因列表顺序差异产生不同股票池。

## 北交所支持

- 北交所代码统一标准化为 `.BJ`，不再错误标记为 `.SZ`。
- 个股评估输入 4、8、92 等北交所代码时，正确识别交易所和板块。
- 数据源不支持某只北交所股票时，按真实数据失败处理，不将其伪装为深市股票或生成占位数据。

## 质量门槛与 manifest

manifest 记录：

- 当日上市 A 股目标数量。
- 行情覆盖数量。
- 评分覆盖数量。
- 各交易所和板块覆盖数量。
- 新增、移除、当日更新失败和基线补齐数量。
- 快照文件的 SHA-256。

发布新快照前，行情覆盖和评分覆盖必须等于当日目标集合。任一股票无真实行情且无上一版真实基线时，整份新快照不发布，线上继续使用上一版合格快照。

上市列表本身必须通过基本完整性检查：包含预期交易所、代码不重复、总数未发生无解释的大幅下降。

## 界面与交互

- 公网界面使用“全部当前上市 A 股”口径，不显示固定 3000 股文案。
- 移除“真实数据股票数量”控件，避免用户将全市场模式误改为局部样本。
- 页面显示本次快照的实际总数和各板块数量，不显示估算值。
- 推荐列表继续按全市场综合评分严格降序。
- 个股评估在全市场快照中查找，不再提示“增大真实数据股票数量”。

## 性能和资源

- 公网应用仍只下载精简价格历史、最新评分、推荐和回测结果，不下载构建专用的全量数据文件。
- 每日构建使用上一版全量历史和 45 天增量更新，避免每次重拉所有股票 4 年数据。
- 市场数值列和模型输入使用合适的紧凑数值类型，减少全市场特征帧的峰值内存。
- 如免费 Runner 在不降低股票覆盖的前提下仍无法完成当前集成模型，优先优化数据类型和中间表生命周期，不改成部分股票评分。

## 验证

- 测试上证、深证、创业板、科创板和北交所代码标准化。
- 测试动态目标集合、新上市加入、退市移除、基线补齐和顺序无关性。
- 测试行情或评分少于当日目标数时禁止发布。
- 生成本地全市场快照，确认 manifest 目标数、行情数和评分数一致。
- 公开 Dataset 按应用文件清单下载，确认不下载构建回退文件。
- Hugging Face Space 健康检查、推荐页、各板块覆盖和北交所个股评估通过。
- 手动触发一次 GitHub Actions 全市场更新，成功发布后才认为上线完成。

## 已知限制

- “全部股票”指真实数据源确认的当前上市 A 股，不包含其他证券类型。
- 数据源上市列表延迟、停牌、新股历史过短和当日网络故障会在 manifest 和页面中显示。
- 股票池扩展不代表模型胜率必然提高，原回测门槛和风控规则继续生效。
