# 华策 Alpha 投研平台公网部署设计

## 目标

将现有 Streamlit 应用部署为任何人都可访问的免费公网服务，并使日常推荐页面在应用唤醒后可快速打开。系统仍只使用真实 A 股数据，不引入演示数据，不承诺收益或固定胜率。

## 托管方案

采用公开的 Hugging Face Docker Space，使用免费 CPU Basic 资源运行 Streamlit。Space 代码和应用均公开，不在仓库中保存 Tushare token 或其他密钥。需要密钥时通过 Space Secrets 注入环境变量。

免费 Space 闲置后可能休眠，因此不将它作为定时计算器。定时任务由公开 GitHub 仓库的 GitHub Actions 执行，应用容器只负责读取已生成快照和处理交互。

## 数据与计算架构

### 每日预计算

- 交易日收盘后由 GitHub Actions 触发数据更新。
- 拉取所选四个板块最多 800 只股票的日线、行业和财务数据。
- 生成特征、训练模型、执行样本外回测，并输出当日推荐、评分、资金曲线和数据源状态。
- 计算成功后才替换线上快照，避免半成品或空结果上线。
- 定时更新失败时保留最近一次成功快照，并在页面显示数据日期和过期状态。
- 保留手动 workflow dispatch，便于管理者立即重新生成。

### 快照分层

- `latest_result`：推荐列表、回测指标、数据源状态和生成时间，用于首屏快速展示。
- `latest_scores`：800 股最新特征和模型评分，用于排名、筛选和个股摘要。
- `market_history`：个股评估所需的精简历史数据，按需读取，不在首屏全量载入。
- `manifest`：版本、交易日、生成时间、股票数、文件校验和数据源说明。

大型快照使用 Hugging Face Dataset 仓库保存，应用启动时按 manifest 版本下载。容器内下载缓存只用于当前实例，不假设免费磁盘永久存储。

## 用户请求路径

1. 打开应用时只加载 manifest 和 `latest_result`，休眠唤醒后尽快显示主界面。
2. 点击“生成今日推荐”时读取共享预计算结果，不重复拉取 800 股和训练模型。
3. 输入股票代码时，先在 `latest_scores` 中查找，再按需加载该股历史数据绘图。
4. 如输入股票不在 800 股快照中，允许一次轻量在线补拉；对同一代码使用进程级 TTL 缓存和并发锁，避免重复请求击穿数据源。
5. 访客不能触发全量数据刷新，也不在页面中输入或查看平台级 Tushare token。

## 性能与并发

- 页面不在 Streamlit session state 中复制整个市场数据集，全局不可变数据用 `st.cache_data` 共享。
- 首屏只加载小型结果文件；价格历史在访问个股时延迟加载。
- 大池推荐读取目标为唤醒完成后 3 秒内；个股快照内评估目标为 5 秒内。免费 Space 从休眠状态唤醒的时间不计入该目标。
- 定时拉取使用有界并发、重试和节流，并将失败股票记入 manifest；不以无限并发换取速度。

## 故障处理

- 快照不存在或校验失败：应用不生成伪结果，显示维护状态和最近一次错误摘要。
- AKShare 失败：预计算尝试 Tushare（如已配置）和现有真实数据缓存；快照数据日期不更新时必须标记过期。
- 个股补拉失败：保留页面其他功能，只在该个股区域显示可操作错误。
- 并发用户触发相同资源时，第一个请求执行下载或补拉，其他请求等待同一结果。

## 安全与公开边界

- 代码、Dockerfile、快照结构和应用界面公开。
- Tushare token、Hugging Face write token 和自动发布凭据只保存在托管平台 Secrets 中。
- 不接收任意文件上传，不执行用户提供的代码，股票代码严格限制为合法 A 股代码。
- 页面持续显示“不构成投资建议、不承诺收益”，同时显示数据日期和回测门槛状态。

## 部署交付物

- 可复现的 `Dockerfile` 和锁定的 Python 依赖。
- Hugging Face Space 配置及公开访问 URL。
- Hugging Face Dataset 快照仓库及 manifest 校验。
- GitHub Actions 定时更新和手动更新 workflow。
- 快照生成命令、本地 Docker 启动命令和故障排查说明。

## 验收标准

1. 未登录用户可通过公网 URL 打开应用。
2. 页面不会在首次打开时自动训练，推荐操作读取最新成功快照。
3. 推荐列表严格按综合评分降序，并显示快照日期、覆盖股票数和数据源状态。
4. 快照内个股评估可正常显示评分、走势和风险信息；快照外个股补拉失败不影响全局。
5. 预计算失败后线上仍可使用上一版合格快照，页面明确标记过期。
6. 自动化测试、Python 编译、Docker 构建、容器健康检查和公网 HTTP 访问均通过。

## 已知限制

- 免费 Hugging Face Space 会休眠，不能承诺 24 小时无冷启动延迟。
- 公开免费托管无法保证中国大陆所有网络环境的访问质量。
- AKShare/Tushare 的可用性、频率限制和历史数据完整性不由本系统控制。
- 免费资源适合公开试用和轻中度并发，不对高并发商业 SLA 作保证。
